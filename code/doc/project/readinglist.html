AIPS++ Reading List

This is a reading list for the first meeting of the AIPS++ Scientific and Technical Advisory Group.

Overview of AIPS++

  • The AIPS++ User Specifications describe the desired scientific capabilities of AIPS++.
  • The AIPS++ Mission Statement provides a more concise description of the goals of AIPS++.
  • Brian Glendenning has written an document describing the Design of AIPS++ . Although this is somewhat out-of-date in places, it does give a good overview of the basic design.
  • Brian has also written a paper for ADASS95 on the AIPS++ Experience in Object-Oriented Design.
  • As we start alpha testing of AIPS++, we have written a document on Getting Started in AIPS++. This is useful for the flavor of how users will interact with AIPS++.
  • The Project submits AIPS++ Quarterly Reports to the AIPS++ Executive Committee. These reports provide information on the progress of AIPS++ over the last 18 months.
  • Other information about the project may be accessed through the AIPS++ Home Page.

Synthesis processing

  • In mid 1995, the project decided to adopt the Hamaker-Bregman-Sault measurement equation as the basis for synthesis (and also some single dish) processing. The design of the synthesis system is described in a memo by Tim Cornwell and Mark Wieringa: Design of Synthesis Calibration and Imaging
  • A demonstration of the use of the AIPS++ MeasurementEquation in Synthesis processing is available.
  • A system for synthesis processing is now in alpha testing, mainly at the AOC. The users guide is available online: Guide to Synthesis Processing

Single Dish processing

  • AIPS++ is committed to providing single dish analysis capabilities for the Green Bank Telescope. The first deliverable in this is a program called SDCalc targeted towards starightforward interactive analysis of spectra and time-series. A useful summary of design documents can be found on Bob Garwood's home page
  • Bob has also written a paper for ADASS95 on the AIPS++ and the GBT.

The AIPS++ Library

  • Coding in AIPS++ is described in the AIPS++ Programming Manual.
  • By fiat, all data in AIPS++ are stored in the AIPS++ Table System. See also the proposed extensions to the Table Query Language
  • We have developed AIPS++ Arrays for the storage and manipulation of arrays.
  • Quantities with units, coordinate systems and reference frames are handled using the AIPS++ Measures System written by Wim Brouw.

Glish/User interface

  • The AIPS++ user interface, both command line and graphical, is through the Glish system, originally written by Vern Paxson of Lawrence Berkeley Laboratory (vern@ee.lbl.gov) but now being maintained by Darrell Schiebel. The capabilities of Glish are described in the Glish Manual (also in postscript)
    • .
  • Darrell has also written a paper for ADASS95 on the Programming in Glish.
  • Rick Fisher has written a very useful Glish tutorial
  • NRAO's recent work on and plans for Glish are described in the Glish development plan.

Tasking etc

A document on the Tasking Environment of AIPS++ describes the overall run-time system that scientists see. In addition, there is a draft (but nearly complete) document on How to code AIPS++ applications.

Plans for parallel processing

Doug Roberts of NCSA has written a document describing a Plan for Parallelization of AIPS++, (also in postscript). This analyzes the scientific specifications for AIPS++ and how they are best to be implemented on parallel machines.

Visualization

The strategy of AIPS++ towards visualization has evolved over the last year. Our straightforward image viewing needs are to be met by the AIPSView program. See:
  • AIPSView design document
  • AIPSView User's Manual
  • There is also a paper for ADASS95 on the AIPSView astronomy visualization tools.
For the development of complicated visualization algorithms, Tom Oosterloo (ATNF) and John Pixton (NCSA) are working on an Image Display Library.

Measures

The handling of measurements in AIPS++ is based upon the AIPS++ Measures System developed by Wim Brouw. This system is designed to handle the specification and processing of quantities with units, coordinate systems, and reference frames.

Documentation

  • It is our intention that all documentation be served and maintained via the WWW. See the Prototype Documentation web page
  • Programmer documentation is mostly written into the C++ code. These are extracted and converted into html using a preprocessor: cxx2html. Examples are found in the aips/implement area.
  • Information on how to code in AIPS++ is available from the AIPS++ Programmer's Reference Manual
  • User's documentation is written in Latex. Glish-based functionality is described in the AIPS++ User Reference Manual

VLBI Processing

Tony Beasley has written a document describing Specifications for VLBI processing in AIPS++. On Oct 23, there will be a meeting in Madrid, Spain to discuss further VLBI specifications for VLBI in AIPS++. Athol Kemball will report on this meeting.

Parkes Multibeam Project experience

The Parkes 21cm Multibeam Receiver is an upgrade to the Parkes Telescope operated by the Australia Telescope National Facility (ATNF). The upgrade consists of a 13-beam cooled 21cm receiver system, which will be located at the prime focus of the 64m dish. The project has adopted AIPS++ as the basis for its software (see the software plan).

Plans for Beta and First releases

The AIPS++ project plans to make a beta release in early 1997, followed by a first release some 6 months later (see the document describing Planning for Beta and First releases).
